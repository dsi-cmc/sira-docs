# (POST: /api/reportes/mapa-auxiliar/update)

**Descrição:**  
Permite a actualização de registos já submetidos num determinado trimestre. Ao contrário da submissão inicial, este serviço permite enviar apenas os blocos de dados que necessitam de correcção (ex: apenas participantes ou apenas imóveis).

---

## Contexto de utilização

- Utilizado para corrigir erros detectados após a submissão inicial.
- Deve ser usado quando a entidade supervisionada identifica discrepâncias nos dados já enviados à CMC.

**URLs de referência de modelo (disponíveis em /exemplos):**

- Modelo JSON de Actualização: [/exemplos/actualizar-mapa-auxiliar-modelo.json](/exemplos/actualizar-mapa-auxiliar-modelo.json)

## Regras de negócio relevantes

- **Identificadores Únicos:** A actualização baseia-se em chaves únicas para localizar o registo existente no sistema:
  - **Participantes:** Localizado pelo `nif` do participante.
  - **Activos Financeiros:** Localizado pela `conta` ou identificador do activo.
  - **Activos Imobiliários:** Localizado pelo `numero_imovel`.
- **Flexibilidade:** Pode enviar um, dois ou os três blocos de dados (`participantes`, `activosFinanceiros`, `activosImobiliarios`). O sistema só processará os arrays que forem incluídos no JSON.
- **Data de Referência:** O `ano` e `trimestre` são **obrigatórios** no topo do objecto para identificar o período a actualizar.
- **Penalidade por Mora:** Se a actualização ocorrer após o prazo regulamentar (conforme regra de `LOCK_DAY`), o status de resposta será `ACTUALIZADO COM PENALIDADE`.

---

## Propriedades do Serviço

| Nome            | Valor                                                        | Descrição                           |
| --------------- | ------------------------------------------------------------ | ----------------------------------- |
| Método HTTP     | POST                                                         | Actualização de dados existentes    |
| Cabeçalho       | Content-Type: application/json                               | Formato JSON                        |
| Autorização     | Bearer Token                                                 | Obrigatório                         |
| URL Produção    | https://reportes.cmc.ao/api/reportes/mapa-auxiliar/update    | Endpoint oficial de actualização    |
| URL Homologação | https://reporteshml.cmc.ao/api/reportes/mapa-auxiliar/update | Ambiente de testes                  |

## Exemplo de Requisição (Actualizar apenas Participantes)

```json
{
  "ano": 2025,
  "trimestre": 3,
  "participantes": [
    {
      "nif_fundo": "000000000",
      "nome_fundo": "ZPX",
      "categoria_up": "Categoria A",
      "nome": "João Manuel Silva - Atualizado",
      "nif": "5001234567",
      "dt_subscricao": "2025-09-15",
      "valor_nominal_up": 1050.00,
      "quantidade_up_subscritas": 50,
      "valor_subscrito": 52500.00,
      "valor_liquidado": 52500.00,
      "valor_actual_up": 53000.00,
      "nivel_risco": "Baixo"
    }
  ]
}
```

```json
{
  "sucesso": true,
  "mensagem": "Mapa Auxiliar actualizado com sucesso.",
  "dados": {
    "status": "ACTUALIZADO"
  }
}
```

> **Nota:** Caso a actualização seja efectuada após o prazo regulamentar, o campo `status` retornará o valor `"ACTUALIZADO COM PENALIDADE"`.
