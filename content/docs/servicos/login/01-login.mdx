# (POST: /auth/login)

**Descrição:**  
Permite que o utilizador se autentique no sistema, enviando as suas credenciais (e-mail e password) para obter uma chave de acesso (token). Esta chave é obrigatória para aceder aos outros serviços da API protegidos.

---

## Contexto de utilização

- Destinado a entidades supervisionadas e utilizadores autorizados para envio ou consulta de reportes.
- Deve ser utilizado sempre que um novo token é necessário, como na primeira autenticação ou após a expiração do token anterior.

---

## Regras de negócio relevantes

- **Expiração:** O token retornado possui um tempo de vida configurado no servidor (geralmente 1 hora).
- **Cabeçalho de Autorização:** O token deve ser enviado em todas as requisições subsequentes no cabeçalho `Authorization: Bearer <token>`.
- **Tentativas de Login:** São permitidas até **3 tentativas** consecutivas. À 4ª tentativa falhada, a conta é automaticamente bloqueada por segurança.
- **Utilizador Bloqueado:** Caso a conta esteja bloqueada, o utilizador deverá contactar o administrador do sistema para o desbloqueio.
- **Rate Limit:** Existe uma protecção contra ataques de força bruta que limita a frequência das tentativas de login.

---

## Propriedades do Serviço

| Nome            | Valor                                 | Descrição                                 |
| --------------- | ------------------------------------- | ----------------------------------------- |
| Método HTTP     | POST                                  | Envia dados ao servidor para autenticação |
| Cabeçalho       | Content-Type: application/json        | Define que o corpo é JSON                 |
| URL Produção    | https://reportes.cmc.ao/auth/login    | Endpoint de login                         |
| URL Homologação | https://reporteshml.cmc.ao/auth/login | Endpoint de teste                         |

---

## Campos de Entrada

| Nome     | Descrição                   | Tipo   | Preenchimento | Exemplo                  |
| -------- | --------------------------- | ------ | ------------- | ------------------------ |
| email    | E-mail do utilizador        | String | Obrigatório   | utilizador@exemplo.com   |
| password | Palavra-passe de acesso     | String | Obrigatório   | SenhaForte123!           |

---

## Exemplo de Requisição

```json
{
  "email": "utilizador@exemplo.com",
  "password": "senhaSegura@123"
}
```

---

## Exemplo de Resposta (Sucesso)

```json
{
  "sucesso": true,
  "mensagem": "Login efectuado com sucesso",
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "user": {
    "email": "utilizador@exemplo.com",
    "name": "Nome da Entidade"
  }
}
```

---

## Exemplo de Resposta (Erro de Credenciais)

```json
{
  "sucesso": false,
  "mensagem": "Credenciais inválidas. Tentativas restantes: 2",
  "erros": []
}
```
